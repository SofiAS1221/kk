План тестирования проекта "Фитнес-трекер"

 1. Цели тестирования
- Проверить соответствие функционала техническому заданию.
- Оценить удобство использования (UX) интерфейса.
- Выявить критические и незначительные ошибки.

2. Типы тестирования
- Функциональное тестирование: Проверка всех сценариев использования.
- Пользовательское тестирование: Оценка интерфейса фокус-группой.

3. Критерии успеха
- Все ключевые функции (CRUD операций, расчет, экспорт) работают корректно.
- Средняя оценка удобства интерфейса от пользователей >= 4.

4. Среда тестирования
- Chrome 120+, Windows 11.
- Node.js v18, база данных SQLite.
